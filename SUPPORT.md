# Support

For support questions on f3d1 ecosystem packages:

| Channel | When to use |
|---|---|
| GitHub Issues on the relevant repo | Bug reports, feature requests with reproducible context |
| GitHub Discussions | Open-ended questions, design feedback, "is this the right tool for X" |
| Email smigolsmigol@protonmail.com | Direct outreach, partnerships, paid consulting inquiries |

For security disclosures: see [SECURITY.md](SECURITY.md). Do NOT file public issues for vulnerabilities.

## Per-package support

Each repo has its own README + examples + verify scripts:

- [f3dx](https://github.com/smigolsmigol/f3dx#readme) - drop-in for openai/anthropic SDKs
- [llmkit](https://github.com/smigolsmigol/llmkit#readme) - hosted gateway docs
- [f3dx-cache](https://github.com/smigolsmigol/f3dx-cache#readme) - cache + replay docs
- [pydantic-cal](https://github.com/smigolsmigol/pydantic-cal#readme) - calibration metrics
- [f3dx-router](https://github.com/smigolsmigol/f3dx-router#readme) - in-process router
- [f3dx-bench](https://github.com/smigolsmigol/f3dx-bench#readme) - bench dashboard
- [tracewright](https://github.com/smigolsmigol/tracewright#readme) - trace replay adapter
- [keyguard](https://github.com/smigolsmigol/keyguard#readme) - security linter

## Response expectations

Issues / discussions: 7 days for first response, faster for security.
PRs: 7 days for first review.
Quiet weeks happen. If something is genuinely urgent, email.
